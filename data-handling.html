Data handling

Belief graphs, dialogue notes, confidence updates, and commitment records can be sensitive. Normativity demos should default to local-first handling unless a page explicitly explains sync, storage, retention, and deletion.

Current rule

If a tool does not clearly describe where data goes, treat it as a local demo and avoid entering sensitive real-world information.

Handling matrix

Surface Data involved Default handling Required before sync or publication
Public research pages Page visits and ordinary browser requests. No first-party analytics declared on static pages. Publish analytics provider, retention, opt-out, and purpose.
WRE demos Judgments, principles, background theories, tensions, and repair notes. Local-first demo storage and user-triggered export. Consent, retention schedule, encryption posture, deletion path, and review owner.
Dialogue examples Arguments, confidence changes, response text, and participant notes. Labeled synthetic or illustrative examples on public pages. Participant consent, redaction review, appeal path, and removal policy.
Contact requests Email address, message content, and requested follow-up. Handled through the sender's email provider and the receiving inbox. Named data owner and published deletion/correction process.

Participant controls expected in production

Export

Machine-readable export for belief graphs, notes, confidence updates, and commitments.

Delete

Clear deletion for local data and a documented request path for any server-side records.

Revoke

Ability to retract public visibility or follow-through plans without public shaming.

Review

Human review for contested records, moderation decisions, and correction requests.

Redact

Pre-publication redaction for personal details, third-party claims, and sensitive beliefs.

Port

Durable local files so users are not locked into a prototype or pilot service.

Normativity treats sensitive belief and dialogue records as high-trust data.
