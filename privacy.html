Privacy and data use

Normativity's public demos should be treated as local-first pilots unless a page explicitly states that server sync is enabled. Sensitive belief, dialogue, and commitment data should be minimized, exportable, and deletable.

Plain-language rule

Do not enter sensitive information into a demo unless the page explains what is stored, where it is stored, how long it is kept, and how to export or delete it.

Data inventory

Dialogue or writing examples Labeled example data on public pages. Example data is not a real user record.
Contact emails Your email provider and the receiving Normativity inbox. Request deletion or correction by email.

Participant controls

Export

Tools that store belief or session data should expose a machine-readable export.

Delete

Local demos should provide a clear local reset or deletion route.

Review

Commitment records should support review, redaction, revocation, and appeal.

Retention stance

Production retention periods are not yet published because the public site is presenting research and labeled demos, not a production participant database. Any production pilot should publish a retention schedule before collecting sensitive data.
